### The disease, fixed

MCL resolved to `202608` — a contract that last traded **2026-07-21**. Resolution now
computes a real last-trade **date** per contract family instead of comparing `YYYYMM`
strings with a `cutoff_day <= 20` heuristic. MCL resolves to `202609`.

The energy rule computes MCL Aug-2026 last trade = **2026-07-21**, reproducing the
observed bar-data gap exactly (MCL alone stopped at 07-21; all nine peers reached
07-22). That is independent validation, not a self-consistent assertion.

`return schedule[-1]` — which silently handed back an expired month once the
hand-written list ran out — is gone. Exhaustion raises `ExpiryScheduleExhausted`;
the provider degrades to `None` (symbol skipped and reported), never a stale month.
Because months are now generated from a per-symbol cycle, the dated cliffs
(SIL ~Dec 2026 first) no longer exist as a class.

### Two defects the tests caught mid-build

Both were in work I had just written, and both would have shipped silently:

1. **Median cohort reference (W6A-3).** A median staleness baseline is dragged along
   by the stale symbols themselves — if several symbols go dark together the median
   moves with them and the lag vanishes precisely when it matters. Changed to the
   group **minimum** (the freshest symbol states what was achievable this run).
   Pinned by `test_stale_majority_cannot_hide_a_laggard`.
2. **A flag incapable of being true (W6A-4).** `ROLL_BUFFER_DAYS` (5) equals
   `ROLL_WARNING_DAYS` (5), so a roll warning computed off the *resolved* contract
   could never fire — resolution always steps forward before the window opens. That
   is the "max_hold can never fire" class: evidence that looks like coverage and is
   structurally dead. Warning/critical/block are now computed off
   `shadow_front_expiry`, the nearest still-trading contract. Pinned by
   `test_shadow_block_flag_is_capable_of_being_both_values`.

A third was in my own test suite: the first draft of the bars-state tests read the
wall clock — the same defect class W6A-5 exists to convert. `today` is now injectable.

### D-decisions as instructed

- **D1** — the unit fails only if **all** symbols fail (or nothing was attempted, or
  IBKR is dead → exit 2). Partial failure is never silent: `bars_refresh_state.v1`
  plus a `BARS_REFRESH_STATE` marker line are written unconditionally, carrying
  per-symbol status, `last_bar_utc`, `age_days`, and `age_vs_cohort_days`. The
  cohort-relative field is the one that actually caught MCL (age 2 against a cohort
  of 1); an absolute threshold would not have.
- **D2** — shadow/warn-only. Verified against the production
  `runtime/futures_roll_state.json`: **zero live-field diffs across all nine symbols
  and an identical summary block.** The blocking flip is its own PA.
- **D4** — Lane B owns `chad-xgb-train`. Not touched here.
- **SIL/M6E** — SIL stays **out** of the tradable/gated universe (data optional); it
  is not published to the roll calendar, so no gate covers a ghost. Its resolution
  cycle is restricted to liquid Mar/May/Jul/Sep/Dec months. Volume evidence cited
  in-code at both sites: the Aug-2026 serial the old resolver picked ran a median
  20-day volume of **179** contracts against MGC's **107,654** — ~600× thinner.
  M6E is unchanged: `omega_macro` consumes it, so it keeps full coverage.

## Open follow-ups

1. **`EXPIRY_SCHEDULE` compat shim** (`futures_contract_resolver.py`) — a derived,
   `DEPRECATED`-marked dict computed at import from `now()`. A repo-wide search finds
   **zero code readers** (only comments and an SSOT doc). Removing it would drop an
   import-time clock dependency from a module whose whole purpose is determinism.
   Left in place because deleting an exported public name is the operator's call.
2. **Holiday calendar.** Business days are Mon–Fri; exchange holidays are not modelled,
   so a computed last-trade date can run up to ~2 days **late** (worked example in the
   module docstring: MCL Jun-2026 computes 05-20 vs IBKR truth 05-18).
   `ROLL_BUFFER_DAYS=5` exceeds that error, so resolution rolls early, never late.
   **Do not lower the buffer without adding a holiday calendar** — stated in-code.
3. **Continuous back-adjusted series** — deferred at plan stage (D5). M6E/ZB/ZN carry
   88–145 bars because each is a single-contract history. Nothing needs >145 today.
4. **`bars_refresh_state.v1` sentinel row** — the artifact is written but no
   `config/exterminator.json` feed row consumes it yet. That is a config change and
   therefore a Pending Action; it also overlaps Lane B's EXS7 adoption work, so it
   should land there rather than be duplicated here.
